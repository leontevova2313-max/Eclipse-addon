# Contributing

## Как вносить изменения

1. Создай отдельную ветку от `main`.
2. Держи изменения локальными и тематически цельными.
3. Не смешивай визуальные правки, сетевые фиксы и крупный рефакторинг в один giant commit.
4. Перед PR прогони локальную сборку и базовую ручную проверку.

## Рекомендуемый формат веток

- `feature/<short-name>`
- `fix/<short-name>`
- `refactor/<short-name>`
- `docs/<short-name>`

## Стиль изменений

- не размазывай магические константы по коду
- не вноси UI-логику прямо в gameplay-модули
- если добавляешь настройку темы, держи её в theme/render слое
- если правишь производительность, опиши в PR что именно уменьшилось: draw calls, allocations, повторные вычисления, дубли рендера

## Перед Pull Request

Проверь:
- проект собирается локально
- нет лишних временных файлов
- нет случайных debug prints
- README / CHANGELOG обновлены, если изменение заметно для пользователя
- скриншоты обновлены, если сильно менялся UI

## Что желательно описывать в PR

- что поменялось
- почему это было нужно
- какие файлы затронуты сильнее всего
- какие риски остались
- как это тестировалось локально
